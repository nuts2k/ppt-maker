# M2 评测体系与路线决策

## Goal

用一套真实商务 PPT（PPSO 投标方案，~25 页）作为评测集，批量走通 M1 核心链路（OCR → 分类 → mask → clean plate → PPTX），量化各阶段质量，建立失败模式目录，形成是否进入 M3 的书面决策。

## Background

- M1 已交付完整单页链路和结构化报告（`SlideReport`），覆盖发现/分类/mask/自动检查/人工接受/Provider 调用/人工耗时。
- M1 仅在 1 张合成 fixture + 1 张真实页面上验证过，样本不足以暴露系统性失败模式。
- ROADMAP 要求 M2 用真实样例量化核心链路，进入 M3 的条件基于评测结果。
- 用户提供了 PPSO 投标方案的完整内容设计提示词（保存在 `research/ppso-original-requirement.md`），可直接生成 ~25 张真实商务页面。

## Requirements

### R1. 页面图片生成

- 基于 PPSO 原始需求，设计 ~25 页的逐页内容描述和图片生成提示词。
- 原始需求 15 页结构扩展为 ~25 页：拆分重型页（+3）、增加议题过渡页（+4）、补充细节页（+3）。
- 页面类型覆盖：深色封面/结尾、议题引导/过渡页、蓝图/架构图页、结构化内容页、时间线/矩阵页。
- 使用 gpt-image-2 生成接口，固定 `size: "2048x1152"`、`quality: "high"`、`output_format: "png"`。
- 每页独立提示词，视觉规范（色值、布局、字体）作为固定前缀嵌入每条提示词。
- 逐页内容描述和生成提示词必须保存到任务目录，供后续复用。
- 严禁出现任何具体客户名称，统一使用"某大型连锁企业"等通用表述。

### R2. 批量转换

- 先批量生成全部图片，人工筛选后再批量跑转换。
- 每张图片独立走 M1 的 `slide` 链路：init → ocr → review → assist-review → validate-review → mask → clean → pptx。
- clean 和 assist-review 阶段需要 `OPENAI_API_KEY` 和用户显式确认。
- 转换失败的页面不阻断其他页面。

### R3. 评测框架

三层评测：

**逐页自动检查**：沿用 M1 已有的 PPTX 检查（ZIP/XML/16:9/文字/字体/形状）和 clean plate 检查（尺寸/文字残留/mask外差异/容器环），自动 pass/fail。

**逐页人工标注**：每页标注漏字数、错字数、分类错误数、容器破坏数、对象符号误删数。

**全集汇总决策**：按页面类型分组统计通过率，形成失败模式目录。

关键指标：
- 目标文字召回率：被正确发现的目标文字块 / 页面中实际的目标文字块。
- 内容正确率：OCR+校正后文字内容与图片实际文字的一致比例。
- 分类准确率：layout_text vs object_integrated_symbol 分对的比例。
- clean plate 完整率：容器未被破坏的页面占比。
- 端到端通过率：完整走通且人工验收通过的页面占比。

### R4. 路线决策

- 进入 M3 的建议门槛：端到端通过率 ≥ 80%，且失败页面的失败原因可归类且有改进方向。
- 必须形成失败模式目录（按阶段、按页面类型归类）。
- 必须形成书面决策：继续 M3 / 增加技术实验 / 收紧支持边界。
- 门槛和指标可在评测过程中根据实际情况调整。

## Acceptance Criteria

- [ ] ~25 张 PPSO 页面图片已生成，覆盖深色/浅色背景、多文字/少文字、图示/表格/纯文字等类型。
- [ ] 逐页内容描述和生成提示词已保存到任务目录。
- [ ] 全部图片已跑完 M1 转换链路（允许部分阶段失败）。
- [ ] 每页有结构化 `SlideReport` 输出。
- [ ] 每页有人工标注结果（漏字/错字/分类/容器/符号）。
- [ ] 全集汇总报告包含按类型分组的指标统计和失败模式目录。
- [ ] 书面路线决策已形成。

## Technical Notes

- gpt-image-2 生成接口直接支持 `2048x1152`（在 2K 阈值以下，稳定产出）。
- 与 M1 clean plate 编辑接口使用完全相同的尺寸和格式参数。
- 已有 OpenAI Image Provider 和类型契约，只需扩展生成（非编辑）接口。
- 批量工作流可通过脚本串联已有 CLI 命令实现，不需要新框架。

## Out of Scope

- 多页 PPTX 合成（M3 范围）。
- 桌面 UI（M4 范围）。
- 内容策划自动化（M5 范围）。
- 非 16:9 页面支持。
- 评测集以外的页面生成。
